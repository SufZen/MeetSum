# Project Context

# project_name: "MeetSum"
# type: software

## Tech Stack

# language: TypeScript (strict mode)
# framework: Next.js 16.2.6 (App Router, Turbopack)
# database: PostgreSQL (via pg, raw SQL migrations)
# frontend: React 19, shadcn/ui, Tailwind CSS, Recharts
# hosting: Hetzner CPX32 VPS, Docker Compose (prod)
# ci_cd: GitHub Actions (lint, test, typecheck)
# queue: BullMQ on Redis
# storage: MinIO (S3-compatible)
# ai: Google Gemini (gemini-3.5-flash), local Whisper (ivrit-ai)

## Conventions

# naming: camelCase for variables/functions, PascalCase for types/components, kebab-case for files
# testing: Vitest, co-located in tests/ directory, `satisfies` for type-safe fixtures
# git_branching: feat/v{version}-{feature-slug} branches, merge --no-ff to main
# commit_style: "feat:", "fix:", "docs:", "test:", "chore:" prefixes with scope
# code_style: ESLint + Prettier, no semicolons in templates, functional components

## Architecture Decisions

# - Modular monolith: app + worker share types, no microservices
# - Provider adapter pattern: AI providers implement a common interface
# - Repository pattern: meetingRepository abstracts storage backend (json-file or postgres)
# - Audit-first: all external-impact actions must call recordAuditLog()
# - Rate limit presets: defined in lib/rate-limit.ts, applied in middleware
# - Summary templates: addendums to base prompt, not replacements
# - Room suggestions: title prefix extraction + tag-based analysis

## Implementation Rules

# - All new API routes must use requireAppAccess() guard
# - All new tests must use `satisfies` for type-safe fixtures against actual types
# - Never expose raw API keys, media URLs, or internal IDs on public routes
# - All database changes must be additive migrations (no drops/renames)
# - Worker jobs must be idempotent and safe to retry
# - All UI state changes must provide user feedback (toast, loading state)

## Anti-Patterns (never do these)

# - Don't import from 'next/server' in lib/ files (keep lib framework-agnostic)
# - Don't use process.env directly in components (use lib/settings or server-side only)
# - Don't add new AI providers without implementing the full AIProvider interface
# - Don't commit .secrets/, .env.local, or audio samples
# - Don't use `any` type — use `unknown` with type narrowing
# - Don't call external APIs from the render path — use API routes
